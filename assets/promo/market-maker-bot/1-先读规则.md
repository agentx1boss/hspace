# 先读规则:Community Maker 到底在奖励什么

这份资源包是给评论区里问「全自动挂单吗」「怎么做」「稳不稳」的人看的。先说结论:

**自动挂单不是自动赚钱脚本。** 它只是让程序按规则在订单簿两边放限价单。能不能累计 Maker Hours, 能不能拿到 DUSD 或 token 奖励, 取决于平台当期规则、订单是否合格、行情波动、你的风控和执行稳定性。

本文只做学习路线整理, 不构成投资建议, 不提供 StandX 专用 bot, 不承诺收益。

## 一句话理解 SIP-5A

StandX 的 [SIP-5A: Community Maker Yield](https://docs.standx.com/sip/sip-5a-community-maker-yield) 是 SIP-5 的第一阶段。它把原来的 Market Maker Uptime Program 升级成 Community Maker Yield, 目标是把订单簿里的真实流动性变成一个可每日结算的收益层。

简单说, 它不是看你喊了多少交易量, 也不是看你放了一笔钱在那里不动, 而是看你有没有持续提供**真实、可成交、足够靠近价格的双边挂单**。

## Maker Hours 的核心路径

官方规则很长, 新手先记这条路径:

1. 双边挂单:同一个小时里, 买单和卖单都要在订单簿上。
2. 距离合格:挂单要在 mark price 附近的合格区间内。
3. 在线时间:挂单需要维持足够长的时间, 不是闪一下就撤。
4. 数量计分:系统看买卖两边中较小的一边, 并按距离、上限、tier 加权。
5. 形成 Maker Hours:每天按各参与者的 Maker Hours 分配当日奖励池。

官方文档明确写了几个关键点:

- 每个小时需要双边订单, 单边挂单不合格。
- 距离 mark price 越近, 同样数量通常越有价值。
- 每个市场有自己的 Unit Size、单边上限、每小时上限、合格区间和 uptime 门槛。
- 每日结算是主要节奏。
- Quote stuffing、短周期撤单、wash trade、关联账户行为会被排除。

这些参数会随 market/campaign 改变, 所以所有示例都只能帮你理解机制, 不能直接照抄。

## DUSD 和 token 奖励怎么理解

SIP-5A 写的是 flexible reward assets:奖励可以是 DUSD、平台 token allocation, 或两者都有。文档也说明, 在平台 token 尚未发行时, token 部分表现为每日 allocation entries。

DUSD 方面, StandX 的 DUSD 介绍页把它描述为带自然收益的 USD 稳定币, 利息按秒计、周期性分发。但这不等于你做市没有风险。做市的风险来自另一层:订单会被成交, 仓位会变化, 杠杆会放大损益, bot 可能断线或撤单失败。

所以更准确的说法是:

> SIP-5A 奖励的是合格的订单簿流动性贡献, 不是给任何自动挂单无条件发钱。

## 评论区那张图能说明什么

原帖里提到:

- 从 7 月 5 日开始参与。
- 累计 7.09 Maker Hours。
- 获得 16.94 DUSD 收益。
- 配图 quote card 里提到:第一个 SIP-5A 收益已发出, Community Maker 将每天收到 DUSD + token 收益。

这是一条很好的「第一次体感」内容, 但它不是收益保证。它能说明机制已经让普通用户看到了可感知的结算结果, 不能说明所有人、所有参数、所有行情下都会重复同样结果。

## 读完规则后再问的三个问题

1. 我理解「双边挂单」和「被成交后仓位变化」了吗?
2. 我知道当前 market 的合格 band、uptime tier、单边上限和奖励周期在哪里看了吗?
3. 如果 bot 断线、价格突然跳、只成交一边, 我有退出方案吗?

如果三个问题有任何一个答不上来, 先不要上真钱。
