## 2. 问题分级

### P0-A · 准确性硬伤(立即修)

- **FAQ「内容加密传输后存储」**:暗示静态加密,实际 R2 存明文。踩 positioning 用语表护栏(密码门 ≠ 端到端加密,别过度承诺)。
  改:「内容经 **HTTPS** 传输,存储在 Cloudflare 全球边缘(R2);密码只存单向哈希,不存明文」。
- 全站自查:仅 `landing.ts` FAQ 这一处误用;`pages.ts` 隐私页表述已准确。"加密"今后只用于 HTTPS/哈希/Cookie 签名,不用于"内容存储"。

### P0-B · 战略缺口:全中文 vs 开发者画像(决定天花板)

- 定位主画像 = Cursor / Claude Code 开发者,**以英文为主**;获客渠道(Marketplace / Open VSX / MCP / GitHub)也是英文流量为主。**落地页 100% 中文 = 收窄画像后最大的自我设限**,也是英文 slogan「Ship to one, not to all.」终于能当 hero 主标题的实测场。
- 这是 positioning 决议 6 挂账的下一迭代。方案(择一):
  - **A(推荐)**:同一 Worker 出**英文默认版**,`?lang=zh` / `/zh` 出中文;英文 hero 用主 slogan 作 h1。
  - B:按 `Accept-Language` 自动切 + 手动切换器。
  - 工作量:文案抽成中/英两份 map + 语言开关,约半天。

### P1 · 文案对齐 + 视觉强化 + 转化(本轮主体)

**文案对齐(并入 rebranding-plan §5,快、确定性高)**
- **FAQ 免费答案对齐 business-model + 埋克制伏笔**:开头点明「核心能力免费」,末尾埋「未来永久链接 / 自定义品牌 / 团队空间**会**面向 Pro/Team,核心能力永远免费」(用"会面向"非"将推出",留退路)。
- **反定位声明**:差异区 sec-sub 下加一行「我们不是:网盘 · 协作工具 · 建站平台 · 公开画廊」(对齐 positioning §8 边界承诺)。
- **张力句落地**:Hero lead 与 CTA 之间加一行「发布侧匿名,分发侧有回执」(positioning §2 张力句一直没落地)。
- **用词**:功能/体验区凡"册子"仅作叙事比喻保留,产品术语统一"合集"(positioning 用语表)。

**视觉强化**
- **OG 卡过期(重要)**:`/og-card.png` 仍是旧标语「私密分享 AI 生成的内容」,与现 slogan 不一致,社媒分享露出旧信息。按新 slogan(中英)重制。
- **Logo「定向送达」语义**(来自 rebranding-plan §3.1):横杠末端加箭头三角指向圆点,让语义从"横杠+孤立圆点"变成"定向链接→送达目标"。**属品牌资产改动,不擅自定稿**——先出 2–3 个矢量候选投票,涉及 `MARK`/`BRAND_MARK`/`assets/*`/favicon(favicon 缓存顽固,需 `?v=2`)。

**转化打磨**
- **"亲自体验"位置偏低**(第 6 屏):它是最强证据,建议在 Hero CTA 旁加"先体验 demo"次级入口,或整体上移到差异区之前。
- **三支柱差异卡 与 6 功能卡部分重复**(每人一链/回执/版本化):功能卡收敛为只讲差异卡没讲的(Markdown 成刊、文档合集、AI 可直接调用、开源自建)。

### P2 · 锦上添花

- 面向画像的显性身份钩子(如 "Built for Cursor & Claude Code")。
- 增长飞轮:落地页加轻量"看过别人分享而来?"引导,承接页脚署名钩子的流量。
- Hero 前段三个 mock 略密集,可精简或让首屏更快见到"亲自体验"。
- 暗色 accent 微调 `#F0784F → #E87048`(更沉稳,可选)。
- 域名长期换独立短域名(positioning 决议 5;影响密码页/落地页信任感)。

## 3. 建议执行顺序与工作量

| 批次 | 内容 | 工作量 |
|---|---|---|
| **批次 1** | P0-A 准确性修复(FAQ 加密措辞)+ 全站自查 | 15 分钟 |
| **批次 2** | P1 文案对齐(FAQ 免费答案 + 反定位 + 张力句 + 用词)+ OG 卡重制 | ~半天 |
| **批次 3** | P0-B 英文版(方案 A,英文 hero 用主 slogan) | ~半天 |
| **批次 4** | Logo 箭头(先出候选投票)+ 转化打磨(体验入口上移 / 功能卡去重) | ~1 天 |
| **批次 5(择机)** | P2 打磨、飞轮、域名 | 视需要 |

> 顺序说明:批次 1、2 是纯文案/资源、零风险,先落地;英文版(批次 3)是最高战略价值但工作量中等;Logo 是主观视觉,单独走候选投票不阻塞其他。

## 5. 怎么算成功(挂钩商业假设记账口径)

**前置**:落地页当前无埋点。需先接一个隐私优先的轻量统计(Plausible / Umami / Cloudflare Web Analytics,无 Cookie、不与"不被索引"原则冲突),否则下列指标无从观测。

- 英文版上线 → Marketplace/GitHub 来访的中英文占比,验证"画像偏英文"假设
- 体验入口上移 → "亲自体验"点击率(产品理解最短路径)
- OG 更新 → 社媒分享点击率
- 漏斗:落地页访问 → 安装点击 → 首次发布;北极星仍是周活发布者(WAP)

## 6. 明确不在范围

- 不加注册/登录 UI(商业假设未触发,business-model §4)
- 不做公开画廊、不做多文件托管(边界承诺)
- 不引入外部字体/脚本/图片(保持自包含、CSP 安全、加载快)
- 不做三支柱重命名(见 §4 裁定)
