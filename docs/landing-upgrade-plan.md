## 2. 问题分级

### P0-A · 准确性硬伤(立即修,今天)

- **FAQ「内容加密传输后存储」**:暗示静态加密,实际 R2 存明文。踩 positioning 用语表护栏。
  改:「内容经 HTTPS 传输,存储在 Cloudflare 全球边缘(R2)」——不提"加密存储"。
- 顺带自查全站:凡"加密"字样只可用于描述 HTTPS/密码哈希/Cookie 签名,不可用于"内容存储"。

### P0-B · 战略缺口:全中文 vs 开发者画像(决定天花板)

- 定位主画像 = Cursor / Claude Code 开发者,**这群人以英文为主**;获客渠道(Marketplace、Open VSX、MCP、GitHub)也以英文流量为主。**落地页 100% 中文 = 收窄画像后最大的自我设限。**
- 这是 positioning 决议 6 已挂账的"下一迭代"。**建议做成 P0**:英文优先或中英双版。
- 方案(择一):
  - **A(推荐)**:同一 Worker 出**英文版为默认**(`hspace.zhanjian.space` 英文),`?lang=zh` 或 `/zh` 出中文;英文 hero 直接用 **"Ship to one, not to all."** 作主标题(它本就是为英文 hero 设计的,终于有了实测场)。
  - B:按 `Accept-Language` 自动切,带手动切换器。
  - 工作量:landing.ts 文案抽成中/英两份 map + 一个语言开关,约半天。

### P1 · 社交卡过期 + 转化打磨

- **OG 卡片过期**:`/og-card.png` 仍是旧标语「私密分享 AI 生成的内容」,与现 slogan「AI 写完的 demo / Ship to one, not to all.」不一致。分享到 X/Slack 露出的是旧信息。重制 OG 卡(中英各一,英文用主 slogan)。
- **"亲自体验"位置偏低**(第 6 屏):它是全页最强证据(真实产品页),建议上移到 Hero 之后、或在 Hero CTA 旁加一个"先体验 demo"次级入口。
- **三支柱差异卡 与 6 功能卡 部分重复**:差异卡(30 秒/收得回/链接是活的)已覆盖核心;功能卡里"每人一链/回执/版本化"与之重叠。建议功能卡收敛为**只讲差异卡没讲的**(Markdown 成刊、文档合集、AI 可直接调用、开源自建),避免同一卖点讲两遍。
- **缺"反定位"一句**:定位文档有"不是网盘/协作/建站/画廊",落地页没体现。可在差异区加一行小字。

### P2 · 锦上添花

- Hero 前三屏三个 mock(分享卡 + VS Code + 阅读页)略"mock 密集";可考虑合并或让首屏更快见到"亲自体验"。
- 加一句面向画像的显性钩子(如 "Built for Cursor & Claude Code")增强身份认同。
- 增长飞轮:落地页本身可加轻量"看过别人分享而来?"引导(承接页脚署名钩子的流量)。
- 域名信任:长期换独立短域名(与 positioning 决议 5 一致),影响密码页与落地页信任感。

## 4. 怎么算成功(与商业假设的记账口径挂钩)

- **英文版上线** → 观测 Marketplace/GitHub 来的落地页访问中英文占比,验证"画像偏英文"假设
- **体验入口上移** → 观测"亲自体验"点击率(它是产品理解的最短路径)
- **OG 更新** → 社媒分享的点击率(卡片是否对上 slogan)
- 北极星仍是周活发布者(WAP);落地页的职责是把"看到的人"转成"装了插件的人",转化漏斗:落地页访问 → 安装点击 → 首次发布。
