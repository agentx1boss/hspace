# 设计:文档合集(Collections)

> 状态:设计定稿,待排期 · 2026-07-03
> 一句话:把一批各自自包含的 md/html 文档,打包成**一个链接 + 一个密码 + 一个目录页**发出去。

## 0. 定位与红线

- 合集是**"发一本册子"**,不是"部署一个网站"。
- 红线不变:**每个文件必须自包含**。不解析文件间相对引用,不服务 css/js/图片等静态资源,无构建。
- 一切私密语义以**合集为单位**:一个密码、一次过期、一次撤回、一份访问计数。

## 1. 发送方体验(VS Code 插件)

### 入口

1. 资源管理器**右键文件夹** →「HSpace: 发布文件夹为合集」
2. 资源管理器**多选文件**(⌘点选)→ 右键 → 同命令
3. 命令面板 → 选文件夹

### 流程(三步,不能再多)

```
右键文件夹
  → ① 确认清单(QuickPick 多选,默认全选)
       - 只收集文件夹【单层】内的 .md/.markdown/.html/.htm
       - 每行:类型图标 + 文件名 + 提取出的标题预览
       - 底部实时显示:已选 N 篇 / 总大小
  → ② 合集标题(InputBox,预填文件夹名,回车确认)
  → ③ 发布进度 → 完成通知:链接+密码已复制
```

**① 确认清单是必须的,不可跳过。** 这是隐私产品的底线:绝不静默上传一个文件夹的内容——文件夹里可能混着不该发出去的东西。清单让"我在分享什么"在上传前一目了然。

### 排序

- 默认按**文件名自然排序**(`1-intro.md`、`2-body.md`、`10-appendix.md` 顺序正确)。
- MVP 不做拖拽重排(QuickPick 能力限制),在清单副标题提示:"按文件名排序,可用数字前缀控制顺序"。
- 未来:支持 `hspace.json` 清单文件自定义顺序与标题。

### 标题提取

- md:首个 `#` 标题;退回文件名(去扩展名)
- html:`<title>`;退回文件名
- 合集标题:用户在 ② 中确认(预填文件夹名)

### 完成后

- 剪贴板:`https://x7k2m9p.zhanjian.space 密码：4831`(与单文件一致,一次粘贴)
- 「最近发布」面板:合集显示为一项,图标 `$(book)`,描述"合集 · 5 篇";可展开显示各篇(点击直达该篇);改密码/删除等操作挂在父节点上。

### 失败语义

- **全有或全无**:任何一个文件读取失败/超限/被扫描拦截,整次发布失败并指明是哪个文件。不产出"缺了两篇的合集"。

## 2. 接收方体验(核心打磨点)

### 流程

```
收到链接+密码 → 打开(任意一篇的深链也一样)
  → 密码门(整个合集只输一次)
  → 目录页 → 逐篇阅读,篇间自由导航
```

### 密码门

- 机制不变(签名 Cookie `Path=/`,天然覆盖全合集)。
- **验证前不展示合集标题**——内容信息不能先于密码泄露。这是刻意决策。
- **修复一个现有细节**:密码提交成功后现在固定跳 `/`;改为**跳回原请求路径**——别人转给你 `/3` 的深链,输完密码应该直接看到第 3 篇,而不是被扔回目录。(此改动对单页面无感知,对合集是关键体验。)

### 目录页(`/`)

- 与阅读页同一设计语言:42rem 版心、同字体栈、亮暗双主题、品牌页脚。
- 结构:合集标题(h1)→ 元信息行("5 篇 · 2026-07-03 分享",无发布者信息)→ 篇目列表。
- 列表行:**弱化的序号(等宽字体)+ 标题**,整行可点、行高充足(≥48px)、hover 时品牌橙箭头浮现、行间细分隔线。
- 不显示文件名、不显示类型徽标——内容为主,技术细节隐藏。

### 篇目页(`/1`、`/2`…)

- **md**:阅读模板 + 合集导航:
  - 顶部面包屑:`← 目录`(链接)+ 合集标题(弱化)
  - 底部 prev/next 卡片:方向 + 相邻篇标题
  - **宽屏(≥1100px)左侧固定侧栏**:篇目列表,当前篇高亮(品牌橙竖条);窄屏自动隐藏,只留面包屑+prev/next。纯 CSS 媒体查询实现,不引入 JS 框架。
  - `<title>` = `篇名 · 合集名`
- **html**:原样服务,不注入任何导航(不篡改用户内容)。返回目录靠浏览器后退。此不对称性写入文档。

### URL 方案

- `/` = 目录;`/1`、`/2`… = 按序篇目。
- 用序号不用文件名:中文/空格文件名的编码噩梦、隐私(文件名可能含敏感词)、稳定简短。MVP 合集不可变,序号即永久地址。

## 3. API 设计

### 发布

```jsonc
POST /publish
{
  "files": [                       // 出现即为合集(2..N 项),与 html/markdown 单字段互斥
    { "name": "1-总览.md",  "markdown": "# 总览\n…" },
    { "name": "2-数据.html", "html": "<!doctype…" }
  ],
  "title": "Q3 方案",              // 合集标题
  "password": "4831",
  "expiresIn": 604800
}
→ 200
{
  "slug": "x7k2m9p",
  "url": "https://x7k2m9p.zhanjian.space",
  "docs": [ { "index": 1, "title": "总览" }, { "index": 2, "title": "数据" } ],
  "passwordProtected": true,
  "expiresAt": "…",
  "editToken": "…"                 // 匿名时返回
}
```

- 数组顺序即篇目顺序(排序是客户端的 UX 职责,服务端只保序)。
- 每项 `markdown`/`html` 二选一,校验规则与单文件完全一致(逐篇扫描)。

### 限制(新增阈值,均入 wrangler.toml)

| 项 | 匿名 | 登录 |
|---|---|---|
| 篇数 | ≤ 10 | ≤ 50 |
| 总体积 | ≤ 1 MB(沿用 ANON_MAX_SIZE_BYTES) | ≤ 5 MB(新 COLLECTION_MAX_SIZE_BYTES) |
| 频率 | 一个合集 = 一次发布 | 同左 |

### 修改与删除

- PATCH:改密码 / 改过期,与单页面一致(匿名限制同样生效)。
- PATCH 内容:**合集 MVP 不可变**(返回 `collection_content_immutable`),owner 也一样——增删改篇目并入"内容版本化"里程碑,避免半吊子语义。
- DELETE:按前缀清理 R2(`pages/<slug>/`)+ 标记删除,整本册子立即失效。

## 4. 存储与路由(零表迁移)

- `pages` 表一行 = 一个合集:`object_key = pages/<slug>/index.json`(后缀 `.json` 即合集标记,延续"后缀即类型"的约定);`filename` 存合集标题;`size_bytes` 存总量。
- R2 布局:
  ```
  pages/x7k2m9p/index.json    # { title, docs: [{name, title, ext}] }
  pages/x7k2m9p/1.md
  pages/x7k2m9p/2.html
  ```
- 服务路径:子域定位合集行 → 密码门(不变)→ `object_key` 以 `.json` 结尾则按路径分发:`/`→目录页;`/<n>`→ 取 `<n>.<ext>` 渲染(md 进阅读模板+导航,html 原样);其余 404。
- 目录页与篇目页均计入合集 `hits`(按篇统计并入"访问回执"里程碑)。

## 5. 边界与已知取舍

- html 篇目无法注入导航(不篡改内容)——接受。
- md 本地图片引用会裂——记入后续"合集附件"(只允许 md 内部引用的图片,不做任意静态服务,红线可守)。
- 匿名合集不可变:发错了删掉重发(反正链接+密码是一次粘贴)。
- 深链分享给第三者:先撞密码门,验证后直达该篇(依赖上面的"跳回原路径"修复)。
- 重名标题:序号天然消歧。

## 6. 里程碑切分

**MVP(v0.2.0)**
后端:files[] 发布、存储布局、目录/篇目路由、阅读模板导航(面包屑+prev/next+宽屏侧栏)、密码门跳回原路径、按前缀删除。
插件:文件夹/多选入口、确认清单、标题确认、面板合集节点(含展开子项)。

**打磨(v0.2.x)**
已读标记(纯 localStorage,不上报)、目录页"继续阅读"、`hspace.json` 自定义顺序。

**依赖联动**
- MCP 接口从第一天就按 `files[]` 设计,单文件是 `files.length === 1` 的特例。
- 访问回执做按篇统计时,复用本设计的篇目结构。
