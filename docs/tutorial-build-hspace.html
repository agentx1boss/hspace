它是什么

把 AI 刚写好的 HTML demo 或 Markdown 文档,变成一个链接 + 密码,只发给该看的人——带访问回执、可即时撤回、链接不变还能迭代。

  • 强制密码是产品身份——不做公开画廊。
  • 没有永久链接:所有链接都会过期(匿名 3 天、登录 30 天/期可续)。
  • 内容页 noindex + 密码门——唯一可被索引的只有落地页。
  • 自包含页面:CSS/JS/SVG 全内联,零外部资源。(这一页就是证据——下面的图、卡片、动画、切换全在文件里。)

架构:一个 Worker 扛下所有

每篇分享都是一个独立子域 <slug>.zhanjian.space,靠一条通配路由 + 通配 DNS 全部接住。橙线是请求的流向:三类域名进同一个 Worker,再落到三种存储。

技术栈全景

⚡运行时Cloudflare Workers

🧮元数据库D1 · SQLite
⏱️计数 / 限流KV
📝Markdownmarked · 边缘渲染
🔐密码 / 签名Web Crypto
🔑登录GitHub OAuth
🚀CLI / 部署Wrangler
🧩编辑器插件VS Code · vsce/ovsx
🤖AI 调用MCP · OpenAPI
💬Claude Code插件 · /share

边缘密码门:无 Cookie → 401 密码页;提交 → PBKDF2 校验 → HMAC Cookie(24h)→ 303 回到原路径。「每人一链」= 同一链接、每人一个密码,按人统计、可单独即时撤销。

A · 账号与一次性授权

wrangler secret put SESSION_SECRET
wrangler secret put GITHUB_CLIENT_ID
wrangler secret put GITHUB_CLIENT_SECRET
  • GitHub OAuth App 需手动创建,回调填登录接口。

E · GitHub Actions 仓库 Secrets

  • CLOUDFLARE_API_TOKEN —— CI 部署 Worker。
  • VSCODE_KEY —— 发 VS Code Marketplace。
  • OPENVSX_KEY —— 发 Open VSX。
  • NPM_TOKEN —— 发 npm(勾 bypass-2FA)。

  • 后端:推 backend/** 到 main。
  • 插件:改版本 → 打 v<x.y.z> tag。
  • MCP:改版本 → 打 mcp-v<x.y.z> tag。
  • Claude Code 插件:改 plugin.json → 推 main。

踩过的坑

  • 子域 DNS 大小写不敏感 → 用小写 base36 slug。
  • .gitignore 掉 .env;别依赖推送保护兜底。
  • CI 发 npm 用勾了 bypass-2FA 的 automation token。
  • npm version 会自动打 tag → 用 --no-git-tag-version。
  • OpenAPI 兼容 GPT Actions:用 nullable: true,别用 type: [..., "null"]。

值得照抄的产品级不变量

没有永久链接 R2 明文存 —— 不说「加密存储」 内容页 noindex + 密码门 自包含页面

一个边缘 Worker + R2/D1/KV,就够把「私密分享 AI 内容」做成产品级。最花时间的从来不是代码,而是那些只能人工做一次的账号与授权。

What it is

Turn the HTML demo or Markdown doc your AI just wrote into a link + password, sent only to the people who should see it — with view receipts, instant revocation, and iteration without changing the link.

  • Mandatory password is the product's identity — no public gallery.
  • No permanent links: every link expires (anon 3 days, signed-in 30 days/term, renewable).
  • Content pages are noindex + gated — only the landing page is indexable.
  • Self-contained pages: all CSS/JS/SVG inlined; zero external assets. (This page proves it — the diagram, cards, animations and toggle all ship inside the file.)

Architecture: one Worker carries everything

Every share is its own subdomain <slug>.zhanjian.space, caught by one wildcard route + wildcard DNS. The orange lines are the request flow: three host types into one Worker, then down to three storage layers.

The full stack

🧮Metadata DBD1 · SQLite
⏱️Counters / limitsKV
📝Markdownmarked · edge-rendered
🔐Password / signingWeb Crypto

The edge gate: no cookie → 401 password page; submit → PBKDF2 verify → HMAC cookie (24h) → 303 back to the original path. Per-recipient links = one link, a password per recipient, per-person stats, instant isolated revocation.

A · Accounts & one-time auth

  • Cloudflare account; npm i -g wrangler → wrangler login.
  • A domain on Cloudflare (wildcard subdomains).
  • GitHub account + repo (code + Actions).
  • VS Code Marketplace publisher (Azure DevOps PAT), Open VSX token, npm account.

  • Fill wrangler.toml: account_id, route pattern + zone_name, resource IDs.

C · DNS

  • Wildcard *.<domain> → the Worker.
  • A landing subdomain (e.g. hspace.<domain>).

D · Worker secrets (never in the toml)

wrangler secret put COOKIE_SIGNING_SECRET
wrangler secret put SESSION_SECRET
wrangler secret put GITHUB_CLIENT_ID
wrangler secret put GITHUB_CLIENT_SECRET

E · GitHub Actions repo secrets

  • CLOUDFLARE_API_TOKEN — CI deploys the Worker.
  • VSCODE_KEY — VS Code Marketplace.
  • OPENVSX_KEY — Open VSX.
  • NPM_TOKEN — npm (bypass-2FA token).

F · Releases

  • Backend: push backend/** to main.
  • Extension: bump version → tag v<x.y.z>.
  • MCP: bump version → tag mcp-v<x.y.z>.
  • Claude Code plugin: bump plugin.json → push main.

Gotchas

  • Subdomain DNS is case-insensitive → use lowercase base36 slugs.
  • .gitignore your .env; don't rely on push protection.
  • CI npm publish needs a bypass-2FA automation token.
  • npm version auto-tags → use --no-git-tag-version.
  • OpenAPI for GPT Actions: use nullable: true, not type: [..., "null"].

Product invariants worth copying
