# 设计:保存(读者收藏 / 另存副本)

> 状态:一期 ✅ 已实现(代码 + 本地验证),待迁移 D1 + 部署;二期 📋 概要待一期数据(2026-07-06)
> 一句话:读者把收到的稿一键保存进自己账号,存过的稿点开就读——一期收藏(引用),二期另存副本(快照)。

## 目标与定位

读者收到一份稿,读完想留住它。今天的选择是收藏浏览器书签(到期即死链)或 Cmd+S(脱离产品)。「保存」让读者登录后一键把稿存进自己的 HSpace 账号,在 Console 随时翻看。

这是**增长飞轮的下一环**:看稿 → 保存(登录)→ 自己发稿。会点「保存」的读者(收 demo 的开发者/技术 PM)就是主画像本身,不触发外扩闸门(positioning §7)。它也是 §8「匿名轻量、鼓励登录」策略在**读者侧的镜像**:匿名阅读一切照旧,保存是登录才解锁的增益。

**排期约束**:排在两个上量闸门(正式内容扫描 + 举报下架流程)之后——保存是上量后才有复利的功能,闸门是上量的前置。

## 核心赌注与 kill/keep 指标

赌注:**读者愿意为保存而登录**。一期用最小代价验证;数据成立才上二期。

- 转化漏斗:`save_click → login → saved`(走现有 `/e` 埋点);
- kill/keep:**saved 读者 → 首次发稿转化率**(飞轮是否闭环);
- 一期完全可撤销:数据不好 = 删一张表、删一个 tab,没有用户承诺遗留。

## 两层设计总览

| | 一期:收藏(引用) | 二期:另存副本(快照) |
|---|---|---|
| 存什么 | 指向源稿的指针 + 钥匙引用 | 内容快照,成为读者自己的一份稿 |
| 发布者开关 | 不需要(只是书签) | 需要,`allowSave` opt-in |
| 撤回/到期/下架 | 完全尊重,列表显示墓碑 | 副本独立于源稿;下架级联可达 |
| 保留策略 | 随源稿生命周期 | 保留至读者删除(活跃保留)+ 配额 |
| 对外链接 | 无新增(打开走源稿链接) | **无可分享 URL**,仅登录后 Console 可见 |
| 回执 | 回访照常计入源稿回执 | 副本上的阅读不回传(故须发布者明示允许) |

**为什么分期**(2026-07-06 评审结论):二期贵的不是代码量(约一期 3–5 倍),而是不可逆承诺——私有副本是一个全新内容模型(无 URL/无密码/登录鉴权/无有效期,与 pages 四个属性全反);`allowSave` 进 OpenAPI 公开契约与三个插件后永久背着;「保留至你删除」是砍不掉的用户承诺。而这些设计假设全部依赖一期回答的赌注。一期上线即可为二期铺路(saves 表天然记录 lineage),间隔可以很短。

## 一期:收藏(引用)

### 语义

- 收藏 = 书签 + 钥匙引用,**不产生**任何新的访问权限或内容副本;
- 源稿撤回/到期/下架后,收藏列表显示墓碑(已到期/已撤回)——墓碑本身是「稿是阶段性的」这一产品身份的复述;
- 读者回访照常经过内容页、照常计入回执与 grants 计数,发布者继续「心里有数」。

### 免密打开:记住钥匙,不是拆门

保存时读者刚通过密码门,签名 Cookie(`crypto.ts` 的 `<slug>.<grantId>.<exp>.<sig>`)已标识了他用的钥匙(grantId,空 = 共享密码)。收藏记录**钥匙引用**,打开时**每次重验**该钥匙对当前稿是否仍有效:

| 场景 | 行为 |
|---|---|
| 稿正常、钥匙未变 | 免密直接打开 |
| 发布者改了共享密码(=撤回) | 回落密码门,要求重输 |
| 发布者撤销了该 grant(定向踢人) | 回落密码门 |
| 稿到期 / 下架 / 删除 | 墓碑 |

不变量:**绝不缓存"已通过"状态,绝不存明文密码**。共享密码存当时的 `password_hash` 值(PBKDF2 结果),重验 = 与 pages 当前 hash 比对;grant 存 `grant_id`,重验 = 查 `revoked=0`。发布者的撤回心智完全不变:改密码/撤 grant 依然收得回所有人,包括收藏过的人。

### 数据模型

```sql
CREATE TABLE IF NOT EXISTS saves (
  owner_id    TEXT NOT NULL,      -- 读者(users.owner_id)
  slug        TEXT NOT NULL,      -- 源稿(天然 lineage,二期快照复用)
  grant_id    TEXT,               -- 保存时所用 grant;NULL = 共享密码
  key_hash    TEXT,               -- 共享密码时:当时的 pages.password_hash 值;grant 时 NULL
  title       TEXT,               -- 冗余存标题,墓碑后仍可显示
  created_at  INTEGER NOT NULL,
  PRIMARY KEY (owner_id, slug)
);
```

### 流程(零跨域会话)

后端是同一个 Worker 服务内容子域与 `hspace.` 域,签发/验证共用 secret:

1. **保存**:密码门通过后渲染的页面(md 阅读页/合集目录/HTML 注入组件)在页脚署名旁内联一个「保存这份稿」链接,href 指向 `hspace.zhanjian.space/console?save=<签名参数>`——签名参数由服务端渲染时签入(含 slug + grantId + 短时效),Console 域验签后引导登录(未登录)→ 写 saves 行 → 落在「已保存」tab。
2. **免密打开**:Console 点击已保存项 → 服务端重验钥匙引用 → 有效则签发**短时效(~60s)、单次使用**的 token,302 到 `<slug>` 子域,密码门认 token 放行并种正常签名 Cookie;失效则直接跳源稿链接回落密码门。token 短命 + 单次,不构成可分享链接。
3. **列表**:Console「已保存」tab,JOIN pages 标状态(正常 / 已到期 / 已撤回 / 已下架→统一显示"已失效",不暴露下架细节)。

### 一期明确不做

- 不存读者密码明文、不做密码代存;
- 不做快照/副本、无发布者开关;
- 内容页不做登录态识别(保存入口是普通链接,跨域会话零工程)。

## 二期:另存副本(快照)—— 概要,待一期数据后定稿

### 前提

一期 kill/keep 指标成立(读者确实为保存而登录,且存在"想留住内容本体"的需求信号,如墓碑项的重访点击)。

### 关键设计(已评审定向)

- **发布者开关 `allowSave`**,opt-in,默认关。语义是"HSpace 是否协助读者留存副本"——**不是技术防拷贝**(读者本可 Cmd+S),文案禁写"杜绝/防止另存"(过度承诺,同 §5 不写"加密分享"之理)。开关需同步 OpenAPI + VS Code 插件 + MCP + Claude Code 插件四个面。
- **副本是新内容模型**:无 slug/无对外 URL、无密码、仅本人登录鉴权在 Console 查看、无有效期。**不产生可分享链接**,故「所有链接都有有效期」不变量全真;想分享 = 从副本重新发布,照常带有效期。
- **保留策略:保留至你删除(活跃保留)**——账户活跃期间一直在;长期休眠(拟 12 个月未登录)先提醒再清理。**对外叙事禁用「永久」二字**(守"别退回永久叙事"红线;"永久"是无限期成本与法律负债)。
- **配额防网盘化**:公开侧用时间守「不做托管」,私人侧用空间守「不是网盘」——免费档小配额(拟 50 份),更大档案空间留作将来付费钩子(内部设计,不公开定价)。
- **下架级联**:源稿 `status='blocked'` 时所有快照副本同步不可见;内容扫描覆盖副本。失去"到期自动清场"泄压阀后,这是唯一补偿机制,运营 SOP(operations.md)需加级联检查。
- **回执**:副本上的阅读不回传源稿——正因发布者失去回执,快照才必须 opt-in。
- 版本化的稿:快照存保存时刻的当前版;合集:整组复制(index.json + 篇目)。

## 红线核对(一次过)

- ✅ 没有永久链接:引用层无新链接;快照层无 URL、token 短命单次;
- ✅ 收得回:钥匙引用每次重验,撤回对收藏人同样生效;
- ✅ 不是网盘/不做托管:配额 + 活跃保留;
- ✅ 匿名轻量:匿名阅读零变化,保存是登录增益;
- ✅ 自包含页面:保存入口内联,不引外部资源;
- ✅ 文案:不说「永久」(用「保留至你删除」)、不说「杜绝另存」、不暗示加密存储。

## 里程碑

**一期(MVP)✅ 已实现**:
- 数据:`saves(owner_id, slug, grant_id, key_hash, title, created_at)`,PK(owner_id,slug)(schema.sql)。
- 令牌:`crypto.ts` 的 `signScopedToken/verifyScopedToken`,scope 前缀隔离 `save`(阅读页→console 收藏凭证,1h)与 `open`(console→内容页免密开门,60s)。**均含 base64 的 `+//=`,进 URL query 前必须 `encodeURIComponent`**(否则 `+` 被解析成空格致验签失败——已在两处 URL 构造点处理)。
- 阅读页:`readingPage/tocPage` 页脚出「＋ 保存这份稿」(md 单页/合集目录/合集 md 篇目;裸 html 篇目未注入,见下)。
- 免密开门:`servePage` 收 `?k=` open 令牌 → 验签 + KV 单次消费(`ot:<sha256>`,120s)→ 种正常 `hs_<slug>` 24h Cookie + 303 回裸路径;失效/已用/伪造/过期一律回落密码门。
- Console:`/console?save=` 收令牌(未登录先 cookie 暂存穿透 OAuth、登录后幂等 upsert 落库);「Saved」卡片渲染状态(saved/expired/no longer available)与移除;`GET /saves/:slug/open`(重验钥匙→签 open 令牌→302)、`DELETE /saves/:slug`;`sclk`/`save` 埋点。
- 收藏结果分流(不谎报):真入库 `?saved=1`、令牌失效/稿已删 `?save_expired=1`(引导重存)、到配额 `?save_full=1`。
- 配额:每人 `SAVE_MAX`(wrangler.toml,默认 200)条,仅新收藏受限(重存幂等不占额度);列表 `LIMIT == SAVE_MAX` 故取满不静默截断。守"不是网盘",更大额度是将来付费钩子。
- 惰性回收:列表渲染时顺手删掉源稿已彻底消失(`deleted` 终态/页面行不存在)的收藏(非阻塞),不占配额、不留死墓碑;过期(可续期复活)与下架(审核态)保留。
- 本地验证(wrangler dev):save 链接编码正确;有效 open 令牌→免密进正文;单次/伪造/过期令牌→回落密码门;配额到顶→`save_full`;deleted 源稿收藏→惰性回收;saves 表 + join 查询通过。
