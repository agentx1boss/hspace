## 2. Rebranding 目标

1. **文案 100% 对齐 positioning.md**——消除 review 里的 P0/P1 文案问题,让落地页成为"单一事实来源"的准确投影
2. **视觉强化"定向送达"语言**——Logo 箭头化、Hero 增加定向动效、组件统一"链接→目标"的视觉隐喻
3. **补上商业模式伏笔**——不收费但不回避,在 FAQ 和功能矩阵里埋"永久/品牌/团队是未来方向"的种子

## 3. 视觉系统升级

### 3.1 Logo:强化"定向送达"语义

**现状问题**:橙横杠穿过双竖条 H,橙圆点在 H 外右侧。圆点与横杠关系不明确——圆点像"掉出来"的东西,不像"目标"。

**升级方向**:横杠末端加箭头三角,指向圆点;圆点略缩小。语义从"横杠 + 孤立圆点"变成"定向链接 → 送达目标"。

**改动范围**:
- `backend/src/landing.ts` 的 `MARK` 常量
- `backend/src/html.ts` 的 `BRAND_MARK` 常量
- `assets/mark.svg` / `assets/mark-white.svg` / `assets/appicon-*.svg`
- `assets/favicon-*.png`(需重新导出)

**代码改动示意**(landing.ts MARK):
```svg
<!-- Before -->
<rect x="13" y="29.5" width="35" height="5" rx="2.5" fill="#F0784F"/>
<circle cx="51" cy="32" r="6" fill="#F0784F"/>

<!-- After -->
<rect x="13" y="29.5" width="31" height="5" rx="2.5" fill="#F0784F"/>
<polygon points="44,27 50,32 44,37" fill="#F0784F"/>
<circle cx="52" cy="32" r="4.5" fill="#F0784F"/>
```

### 3.2 配色:保持不变

当前配色已经很好——暖橙有辨识度,墨色砖有信任感,暖白背景不冷硬。**不动**。

唯一微调:暗色模式下 accent `#F0784F` 略偏荧光,可考虑降到 `#E87048` 让暗色模式更沉稳。但这是 P2,不阻塞。

### 3.3 字体:保持系统字体栈

当前 `-apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC"...` 是正确的——零加载、跨平台、中文友好。**不动**。

### 3.4 组件:新增"定向链接"视觉母题

在 Hero mock 和体验入口里,统一用一个视觉母题:**链接(横杠) + 箭头 + 目标(圆点)**,与 Logo 呼应。

- Hero mock 的"链接和密码已复制"提示,可加一个微型箭头动效(链接 → 目标)
- 体验入口的 `→` 符号,可换成与 Logo 一致的箭头三角

### 4.2 Hero 张力句(新增)

在 Hero lead 下方、CTA 上方,加一行小字张力句:

```html
<p class="tension">发布侧匿名,分发侧有回执。</p>
```

样式:`font-size:13px; color:var(--muted); font-style:italic; margin-top:-16px; margin-bottom:24px`

这行字解决 positioning.md 第 2 节"张力句"从未落地的问题,同时让"匿名 vs 可追溯"的矛盾被正面承认。

### 4.3 反定位声明(新增)

在"不是又一个 HTML 托管"section 的 sec-sub 下方,加一行反定位:

```html
<p class="anti">我们不是:网盘 · 协作工具 · 建站平台 · 公开画廊。</p>
```

样式:`text-align:center; color:var(--muted); font-size:14px; margin-top:-28px; margin-bottom:36px; letter-spacing:.02em`

这行字让"我们不是什么"可复述,与 positioning.md 第 8 节边界承诺呼应。

## 5. 文案对齐清单(逐项 Before → After)

### 5.1 三支柱重命名(P0)

| 支柱 | Before | After | 理由 |
|------|--------|-------|------|
| 1 | 🚀 **30 秒发出去** | 🚀 **能发出去** | "30 秒"可证伪;结果导向 |
| 2 | 🎯 **发错了?收得回** | 🎯 **收得回** | 统一结果导向,去问句 |
| 3 | 🔁 **链接是活的** | 🔁 **能迭代** | 统一结果导向,更直白 |

支柱 1 的描述也要改:
- Before:"零注册、零配置,编辑器或 AI 对话里一键发布,链接+密码一次粘贴。别家的密码保护要付费套餐加一堆配置,这里是默认。"
- After:"零注册、零配置,编辑器或 AI 对话里一键发布,链接+密码一次粘贴。**秒级完成**,别家的密码保护要付费套餐加一堆配置,这里是默认。"

### 5.2 功能矩阵"册子"→"合集"(P0)

| 位置 | Before | After |
|------|--------|-------|
| 体验入口 th | "一本 3 篇的**册子**" | "一本 3 篇的**合集**" |
| 功能矩阵 feature | "Markdown 成**册**" | "Markdown **成合集**" |

### 5.3 FAQ 答案对齐(P0)

**Q: 免费吗?有什么限制?**

- Before:"匿名免费即用:单文件 ≤ 1MB、最长 7 天有效、每天 50 次。禁止钓鱼与恶意内容,违规会被下架。"
- After:"**核心能力现在全部免费**——密码分享、合集、回执、每人一链、版本化,都不设限。匿名即用:单文件 ≤ 1MB、最长 7 天有效、每天 50 次。禁止钓鱼与恶意内容,违规会被下架。**未来永久链接、自定义品牌、团队空间会面向 Pro/Team,核心能力永远免费。**"

改动三点:
1. 开头点明"核心能力现在全部免费"(对齐 positioning.md)
2. 把"匿名免费"改成"核心能力免费 + 匿名即用"(分离两个概念)
3. 末尾埋商业模式伏笔(对齐 business-model,不收费但不回避)

### 5.4 张力句与反定位(P1)

见 4.2 / 4.3,此处不重复。

## 8. 风险与注意事项

1. **Logo 改动影响 favicon 缓存**——浏览器 favicon 缓存极顽固,改完需加 `?v=2` 查询参数强制刷新
2. **OG 卡重新设计**——社交分享场景第一印象,新 Logo + slogan 要同步更新,否则传播时品牌不一致
3. **"册子"→"合集"可能影响已发布内容**——检查代码里是否有已发布的 HTML 里硬编码了"册子"字样(应该在运行时渲染,不影响存量)
4. **商业模式伏笔要克制**——FAQ 末尾那句"未来永久链接...面向 Pro/Team"是伏笔不是承诺,措辞要留退路(用"会面向"而非"将推出")
